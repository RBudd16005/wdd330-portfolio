Accessing Form Elements

Forms are a main component of Google's home page.
Forms are now more frequently processed on the front end before it is sent to a server.

Form Properties and Methods

The form.submit() method will submit the form automatically. This will not trigger the form submit event.

Form Events

The focus event occurs when an element is focused on.
The blur event occurs when the user moves the focus away from the form element.
The change event occurs when the user moves the focus away from the form element after changing it.

Submitting a Form

You can stop the form from being submitted to the URL by using the preventDefault() function.

Retrieving and Changing Values From a Form

Text input element objects have a value property that can be used to retrieve the text inside the field.
Similar results can be produced in modern browsers by using the placeholder attribute in the HTML markup.

Form Controls

Common types of form control are:
input - These are fields, including text, passwords, check boxes, radio buttons, and file uploads.
select - These are menus for drop-down lists of options.
textarea - These are elements for longer text entry.
button - These are elements for submitting and resetting forms.

Radio Button Input Fields

Radio buttons are created using input fields with type='radio'.
Like checkboxes they allow users to check an option as true, but they provide an exclusive choice of options,
so only one option can be selected.

Hidden Input Fields

Hidden fields can be created using input fields with type='hidden'.
These are not displayed by the browser, but have a 'value' attribute that can contain information that is submitted with the form.
They are often used to send information such as settings or information that the user has already provided.
The information in these fields is not a secret, as it’s visible in the HTML, so shouldn’t be used for sensitive data.

File Input Fields

A file input field can be created using input fields with type='file'.
These are used to upload files.

Other Input Types

There are new input types included in HTML5: number, tel, and color.

Form Validation

Form validation is checking whether a user has entered the information into a form correctly or not.
The API works by simply adding relevant attributes to the form fields.
For example, if a field is a required field that must be filled in, all you need to do is add a
'required' attribute to that field and the browser will take care of the rest.

Disabling the Submit Button

A useful technique is to disable the submit button if there are errors on the form.
If the submit button is disabled then no action is taken when it is clicked.
This can be changed by using the disabled property of the button.
