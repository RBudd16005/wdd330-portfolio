C-style syntax used by JavaScript

Example of Javascript:
const message = 'Hello World!'
alert(message)

also: const message = 'Hello World!';alert(message);

Block code is used with '{}' and does not need an ending semi-colon.

There are reserved words for functions in Javascript that cannot be used as variable names:
abstract, await, boolean, break, byte, case, catch, char, class, const, continue, debugger, default, delete, do, double, else, enum, export, extends, false, final, finally, float, for, function, goto, if, implements, import, in instanceof, int, interface
let, long, native, new, null, package, private, protected, public, return, short, static, super, switch, synchronized, this, throw, throws, transient, true, try, typeof, var, volatile, void, while, with, yield

also do not use: undefined, NaN, Infinity

Seven datatypes: String, Symbol, Number, Boolean, Undefined, Null

Typeof will show what the varialbe is:
typeof 'hello'
<< 'string'

const and let are block scope varialbes that only work in the block they are declared in.

Try to avoid Global Scope variables.

Do not start variables with numbers and avoid starting with underscores unless the variable is private, also try to avoid starting with $.

Variable names are case sensitive.

Be consistant with your naming rules.

Variables started with '{}' mean it is a referance and not the value itself:
const c = { value: 1 };
let d = c; // c.value = 1, d.value = 1
d.value = 2; // c.value = 2, d.value = 2

use ' and " as needed if you want a ' or a " in the string:
'it's me' = error
"it's me' = good!
This also works: 'it\'s me'

string.toUpperCase() = change all characters to upper case
string.toLowerCase() = change all characters to lower case
find characters by index: name.charAt(0)
